# ფილოგენეტიკური ხეების ნახვა და ძირითადი ინტერპრეტაცია

IQ-TREE 2 ჩვეულებრივ ქმნის რამდენიმე შედეგის ფაილს. ხის ვიზუალიზაციისთვის ყველაზე მნიშვნელოვანი ფაილებია:

- `.treefile` — მაქსიმალური დამაჯერებლობის (Maximum Likelihood) მთავარი ხე;
- `.contree` — კონსენსუსური ხე ტოტების მხარდაჭერის მნიშვნელობებით.

თუ ანალიზში გამოიყენეთ `-B 1000`, ფიგურის შესაქმნელად ხშირად უმჯობესია ჯერ `.contree` გახსნათ, რადგან მასში bootstrap მხარდაჭერის მნიშვნელობებია დატანილი.

## 1. შედეგების ფაილების პოვნა

გადადით შედეგების საქაღალდეში:

```bash
cd ~/projects/my_phylogeny/results
```

იხილეთ ფაილები:

```bash
ls -lh
```

მაგალითი:

```text
sequences_tree.treefile
sequences_tree.contree
sequences_tree.iqtree
sequences_tree.log
```

WSL-ის მიმდინარე საქაღალდის Windows Explorer-ში გასახსნელად გაუშვით:

```bash
explorer.exe .
```

## 2. რომელი ხის ფაილი უნდა გამოვიყენოთ

### `.treefile`

ეს არის IQ-TREE-ის მიერ ნაპოვნი საუკეთესო Maximum Likelihood ხე Newick ფორმატში.

გამოიყენეთ, როდესაც გჭირდებათ მთავარი ოპტიმიზებული ხე ან bootstrap-ის გარეშე ჩატარებული ანალიზის შედეგი.

### `.contree`

ეს არის კონსენსუსური ხე ტოტების მხარდაჭერის მნიშვნელობებით. იგი ჩვეულებრივ იქმნება, როდესაც გამოყენებულია bootstrap, მაგალითად:

```bash
iqtree2 -s aligned.fasta -m MFP -B 1000 -T AUTO
```

ტოტების მხარდაჭერის საჩვენებლად და შედეგების ფიგურის მოსამზადებლად, უმეტეს შემთხვევაში გახსენით `.contree`.

ორივე ფაილი ტექსტური Newick ფორმატის ფაილია. მათი გაფართოების შეცვლა საჭირო არ არის.

## 3. ხის გახსნა iTOL-ში

iTOL ნიშნავს **Interactive Tree of Life**-ს და გამოიყენება ფილოგენეტიკური ხეების ონლაინ სანახავად და გასაფორმებლად.

1. გახსენით iTOL-ის ვებგვერდი.
2. შედით ანგარიშში.
3. აირჩიეთ **Upload a tree**.
4. მონიშნეთ `.treefile` ან `.contree` ფაილი.
5. ატვირთვის შემდეგ ხე ავტომატურად გამოჩნდება.

IQ-TREE ხეს Newick ფორმატში წერს, რომელსაც iTOL პირდაპირ იღებს.

### iTOL-ში სასარგებლო მოქმედებები

შეგიძლიათ:

- ხის ფორმა შეცვალოთ rectangular, circular ან unrooted ხედად;
- ტოტების მხარდაჭერის რიცხვები აჩვენოთ ან დამალოთ;
- თანმიმდევრობების სახელები დაარედაქტიროთ ვიზუალურად;
- ტაქსონები ფერებით მონიშნოთ;
- დამატებითი metadata dataset-ები დაამატოთ;
- ხე SVG, PDF ან PNG ფორმატში გაიტანოთ.

## 4. ხის გახსნა FigTree-ში

1. გახსენით FigTree.
2. აირჩიეთ **File → Open**.
3. მონიშნეთ `.treefile` ან `.contree`.
4. საჭიროების მიხედვით ჩართეთ node labels, branch labels და scale bar.

FigTree გამოსადეგია ხის სწრაფი დათვალიერებისთვის, rooting-ის შესამოწმებლად და მარტივი ფიგურების შესაქმნელად.

## 5. ხის ძირითადი ნაწილები

### ბოლოები, ფოთლები ან tips

ხის ბოლოებში განთავსებული სახელები შეესაბამება საწყისი alignment-ის თანმიმდევრობებს.

თუ FASTA სათაური იყო:

```text
>Sequence_1
```

ხეზე ბოლო, როგორც წესი, გამოჩნდება სახელით:

```text
Sequence_1
```

### ტოტები

ტოტები ასახავს მოდელის მიხედვით შეფასებულ ევოლუციურ ცვლილებას.

თუ ხეზე branch length-ებია მოცემული, უფრო გრძელი ტოტი ჩვეულებრივ უფრო მეტ შეფასებულ ცვლილებას ნიშნავს. ეს პირდაპირ არ ნიშნავს, რომ ორგანიზმი უფრო ძველია ან უფრო განვითარებულია.

### შიდა კვანძები

შიდა კვანძები წარმოადგენენ მონაცემებიდან შეფასებულ საერთო წინაპრებს.

### კლადი

კლადი არის საერთო წინაპარი და მისგან წარმოშობილი ყველა შთამომავალი ტაქსონი.

## 6. ტოტების მხარდაჭერის მნიშვნელობები

კვანძებთან არსებული რიცხვები ხშირად bootstrap ან სხვა ტიპის მხარდაჭერის მნიშვნელობებია.

მაგალითად:

```text
95
```

ნიშნავს, რომ შესაბამისმა დაჯგუფებამ bootstrap ანალიზში მაღალი მხარდაჭერა მიიღო.

თუმცა bootstrap მნიშვნელობა არ არის იმის პირდაპირი ალბათობა, რომ კლადი „ნამდვილად სწორია“. ის აჩვენებს, რამდენად სტაბილურად მეორდება ეს დაჯგუფება მონაცემების გადარჩევისას გამოყენებული მეთოდის ფარგლებში.

დაბალი მხარდაჭერის მქონე ტოტების ინტერპრეტაცია სიფრთხილით უნდა მოხდეს.

## 7. Rooted და unrooted ხე

IQ-TREE ჩვეულებრივ ქმნის unrooted ხეს, თუ outgroup ან rooting-ის სხვა მეთოდი არ არის მითითებული.

Unrooted ხე გვიჩვენებს ტაქსონებს შორის ურთიერთობებს, მაგრამ არ განსაზღვრავს ევოლუციის მიმართულებას ან უძველეს საერთო წინაპარს.

ხე მხოლოდ ბიოლოგიურად დასაბუთებული outgroup-ით უნდა დააფესვიანოთ.

IQ-TREE-ის ბრძანებაში outgroup-ის მითითების მაგალითი:

```bash
iqtree2 -s aligned.fasta -m MFP -B 1000 -T AUTO -o Outgroup_sequence --prefix rooted_tree
```

`Outgroup_sequence` ზუსტად უნდა ემთხვეოდეს FASTA სათაურს `>` სიმბოლოს შემდეგ.

## 8. ფიგურის ექსპორტი

რედაქტირებადი სამეცნიერო ფიგურისთვის გამოიყენეთ:

- SVG — ვექტორული და ადვილად რედაქტირებადი;
- PDF — ხარისხიანი ვექტორული დოკუმენტი.

უბრალოდ სანახავად ან პრეზენტაციაში ჩასასმელად შეგიძლიათ გამოიყენოთ:

- PNG — რასტრული გამოსახულება.

მაღალი ხარისხის სტატიის ფიგურისთვის SVG ან PDF ჩვეულებრივ უკეთესი არჩევანია.

## 9. ორიგინალი ფაილების დაცვა

არ შეცვალოთ IQ-TREE-ის მიერ შექმნილი ორიგინალი `.treefile` და `.contree` ფაილები.

გამოსაქვეყნებელი ან გაფორმებული ვერსიები შეინახეთ ცალკე სახელებით, მაგალითად:

```text
sequences_tree.contree
sequences_tree_itol_edited.svg
sequences_tree_final_figure.pdf
```

ეს საშუალებას იძლევა ყოველთვის დაუბრუნდეთ პროგრამის თავდაპირველ შედეგს.

## 10. ინტერპრეტაციისას გავრცელებული შეცდომები

არ უნდა ვივარაუდოთ, რომ:

- ერთმანეთთან ახლოს დახატული ორი ბოლო აუცილებლად უახლოესი ნათესავია — მნიშვნელოვანია კვანძების სტრუქტურა და არა ეკრანზე ფიზიკური მანძილი;
- უფრო გრძელი ტოტი ნიშნავს უფრო „განვითარებულ“ ორგანიზმს;
- ხის მარცხენა მხარეს მოთავსებული ტაქსონი უფრო ძველია;
- მაღალი bootstrap მნიშვნელობა ავტომატურად ადასტურებს ბიოლოგიურ ჰიპოთეზას;
- unrooted ხე გვიჩვენებს ევოლუციის მიმართულებას.

ხის ინტერპრეტაცია უნდა ეყრდნობოდეს topology-ს, branch length-ებს, მხარდაჭერის მნიშვნელობებს, გამოყენებულ მონაცემებსა და ბიოლოგიურ კონტექსტს.

შემდეგი სახელმძღვანელო: [ხშირი პრობლემები და მათი მოგვარება](08_troubleshooting.md).
